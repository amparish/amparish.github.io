This is a simple news scraper, using the Cheerio NPM package to pull articles from Pitchfork.com's news section.

This app is a standard Hangman-style game that takes in keyboard input from the user to dynamically update the HTML page when a letter in a given term is guessed correctly or incorrectly.

At the start, the term to be guessed - randomly pulled from a list of 20 musicians I like - is replaced with underscores in place of the letters. As a player guesses letters, correct letters will replace the underscores, dynamically updating the page each time. Incorrect letters are displayed below this, along with a counter with the remaining guesses allowed on the current term. If a player guesses the term correctly, one point is added to their score, and an image of the musician is displayed. If the remaining guesses counter reaches zero, no points are added, and a new random term is selected while the guess counter and used letters reset.


Adoptable is a website that leverages the Petfinder API to allow users to search for adult and senior cats and dogs available for adoption in their area. This concept was born out of an idea from one of the project's other developers, a strong advocate for the adoption of animals who are less likely to be adopted by most people. Our goal with the project was to surface this data in a user-friendly way, with a simple, modern design that emulated popular online dating apps - specifically, OKCupid, which was the inspiration for the the card-style layout of the search results.

My focus in this project was the implementation of the Petfinder API, which proved to have more issues than expected. In most cases, shelters listing animals on Petfinder would supply all of an animal's information - name, age, breed, a photo, et cetera - but in cases where this data was not provided, those null values would produce errors on the page, or cause a different animal's information to be displayed. This error was resolved in some cases with if statements that displayed "Unavailable" in place of the missing content, or in the case of the photos, with a try/catch statement that inserted a placeholder image if one was not provided by the AJAX call.

Truckstop is a concept for a food truck ordering app that utilizes multiple APIs to present a user-friendly and vendor-friendly space to facilitate ordering from local food trucks. Using a MySQL database, truck location info was placed into the Google Maps API in order to display a full map of available locations.

Menu info is also stored in the database, which users can choose to add to a cart. Upon ordering, the Twilio API sends a text message to the food truck operator with all of the customer's order information.
